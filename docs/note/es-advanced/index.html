文档#

文档元数据#

三个必须的元数据元素如下：

  • _index：索引，一个索引应该是因共同的特性被分组到一起的文档集合。
  • _type：文档表示的对象类别，子分区
  • _id：文档唯一标识。创建一个新的文档，要么提供自己的 _id ，要么让 Elasticsearch 帮你生成。

处理冲突#

同样有乐观锁和悲观锁两种处理概念。权威指南介绍了乐观锁：

  • 乐观并发控制

  • 通过外部系统使用版本控制

使用其它数据库作为主要的数据存储，使用 Elasticsearch 做数据检索。
